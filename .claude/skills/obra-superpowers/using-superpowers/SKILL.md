# USING SUPERPOWERS

Source: https://github.com/obra/superpowers

## Core Rule

Invoke relevant or requested skills BEFORE any response or action. Even a 1% likelihood that a skill applies mandates invocation before proceeding.

## Instruction Hierarchy

1. User's explicit instructions (highest priority)
2. Superpowers skills (override default behavior)
3. Default system prompt (lowest priority)

## Critical Practice

These thoughts are red flags — you are skipping a required skill check:
- "This is just a simple question"
- "I need context first"
- "I'll check skills after exploring"

Skills are invoked BEFORE gathering information or exploring codebases, not after.

## Priority When Multiple Skills Apply

Process skills (brainstorming, debugging) come before implementation skills — they determine HOW you approach tasks.

## Common Rationalizations to Reject

- "This is too simple to need a skill"
- "I already know how to do this"
- "The skill won't add anything here"
- "I'll just do it quickly then check"

None of these are valid. Invoke first. Always.
