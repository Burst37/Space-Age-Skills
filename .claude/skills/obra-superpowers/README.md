# obra/superpowers Skills

Source: https://github.com/obra/superpowers (MIT License)

A complete software development methodology for coding agents. Structured workflows and composable skills that enforce quality gates at every stage.

## Installed Skills

| Skill | Trigger |
|---|---|
| `using-superpowers` | Start of every session — establishes skill priority |
| `brainstorming` | Any new feature or non-trivial task |
| `writing-plans` | After brainstorming is approved |
| `writing-skills` | When creating a new skill document |
| `test-driven-development` | Any new feature, fix, or behavior change |
| `systematic-debugging` | Any bug or unexpected behavior |
| `subagent-driven-development` | Executing plans with independent tasks |
| `verification-before-completion` | Before any completion claim |
| `executing-plans` | Implementing a written plan |
| `finishing-a-development-branch` | When development work is ready to wrap |
| `requesting-code-review` | After tasks, before merging |
| `receiving-code-review` | When feedback arrives |
| `dispatching-parallel-agents` | Multiple independent failures/tasks |
| `using-git-worktrees` | Starting any implementation work |

## Key Principle

Invoke relevant skills BEFORE any response or action. Process skills (brainstorming, debugging) before implementation skills.
