# VERIFICATION BEFORE COMPLETION

Source: https://github.com/obra/superpowers

## Core Rule

NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE.

## The Verification Process

Before claiming anything is finished:

1. **Identify** the command that proves your assertion
2. **Execute** it fully and freshly
3. **Examine** the complete output and exit codes
4. **Confirm** whether results support your claim
5. **Only then** state the conclusion with supporting data

Skipping any step is misrepresentation, not efficiency.

## What Does NOT Count as Verification

- Previous test runs
- Partial checks
- Confidence or intuition
- Agent success reports
- Changed code without testing
- "Should work" or "looks correct"

## Red Flag Language — Stop and Verify

If you are about to say any of these, stop and run verification first:
- "Great!"
- "Perfect!"
- "Done!"
- "That should do it"
- "I believe this works"
- "This looks correct"

## Applies To

- All completion claims
- All satisfaction expressions
- All positive work-state assertions
- Before every commit, PR, or task transition

No exceptions.
