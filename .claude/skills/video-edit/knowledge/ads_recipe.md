# Ads Recipe — direct-response shorts (webinar/product funnels)
Research-grounded, 2026-06-10. Applies whenever the input is an AD (folder/name contains "ad",
CTA is a link/date, not subscribe). Differences vs organic shorts are the point of this file.

## The two metrics everything serves
- **Hook rate** = 3s-views / impressions. Target 30-40%+. Decided in seconds 0-3.
- **Hold rate** = 15s-viewers / 3s-viewers. Target 40%+, 60% = strong. Decided in seconds 5-15.

## Structure: PAS (Problem-Agitate-Solution) + proof + 1 CTA
hook (0-3) → agitate the pain (3-12) → the pivot/solution (12-22) → proof/how (22-end) → single CTA.
Our scripts already follow this — the EDIT must mirror it: pain = darker/tighter visuals,
solution = open/bright visuals (freedom b-roll), CTA = text lock-in of date/link ONLY.

## Hook window (0-3s) — hard rules
1. hook_title text visible at frame 0 (rule 4ae) with the KEY BENEFIT or pain, not a vague tease.
2. LEAD WITH MOTION: static talking head = scrolled past. The hook zoom-in (4ap) + flash counts;
   even better when the first b-roll lands 1.5-3s in.
3. 85% watches MUTED — the hook must work with sound off (text carries it).

## Hold window (5-15s) — THE critical edit zone
Force a visual change EVERY ~2 SECONDS in seconds 5-15. A change = b-roll cut, word_pop landing,
zoom punch, text reveal — anything new for the eye. Audit: list all visual events in 5-15s;
any gap >2.5s in that window = add an element. After 15s the cadence may relax to every 4-5s.

## B-roll usage in ads
- Cadence over quantity: ~40-60% of an ad's runtime has SOMETHING over/instead of the speaker.
- B-roll must LITERALIZE the line being spoken (pause-and-read test, rule 4bs) — "can't leave"
  = hands chained to keyboard; "fire yourself" = closing laptop and walking out.
- Full-screen subject-centered 9:16 crops (rule 4ce), 2-3.5s each, never two back-to-back
  without speaker breath (4z).
- Map the emotional arc: pain lines get grind/dark clips, payoff lines get freedom clips.

## Text overlays
- Sound-off first: every key claim either in captions or a word_pop — never neither.
- Text in the middle 60% of frame (platform UI eats top/bottom 20%).
- Max 2 lines, big type. Value props > clever copy.
- CTA text = the literal action + date ("JUNE 23 · FREE", "LINK IN DESCRIPTION") — nothing else
  competes in the last 3s.

## What ads DON'T get
- NO subscribe-bug (rule 4cf).
- No channel-building beats; every element sells the one next step.
- Don't over-polish: native/UGC-feel founder ads convert 2-4x better than slick brand ads.
  Keep the speaker raw, keep cuts punchy, resist template chrome.

## Testing loop
- The HOOK is the #1 test variable: same body + 3 different hooks (we cut VAR_ ads for this).
- After launch: read the retention curve; the drop-off second tells you exactly where a new
  element is missing. Fix that second, not the whole ad.

Sources: hook/hold benchmarks + 2s-cadence (glued.me, adlibrary.com, benly.ai), sound-off 85% +
text-overlay rules (clicks.video, 3playmedia), PAS/VSL structure (thrivethemes, theperformers.io),
UGC conversion data (billo.app, marpipe).
