What This Document Is

This is the permanent offline reference for the Space Age AI Solutions Claude Code session initialization system. Keep it so you can rebuild the hook from scratch if Claude ever loses context or the repo is corrupted.

Every Claude Code session (web, desktop, GitHub Action) fires a SessionStart hook automatically. That hook does two things: (1) immediately injects a structured additionalContext string that tells Claude exactly what to do before answering any task, and (2) syncs all skills from the repo into ~/.claude/skills/ in the background so they're available as slash commands.

The hook was rebuilt from scratch during the session of 2026-06-21 after the v1 hook was producing a generic/stripped context. The v2 upgrade added the full 4-step hard gate, the tier 2 routing matrix, auto-trigger rules, and a more complete infra quick reference. It also fixed background task safety (guards against empty SKILLS_DST, conditional npm install, safe CLAUDE_ENV_FILE check).

How the Hook Works (Mechanics)

Hook Location & Format

The hook lives at .claude/hooks/session-start.sh in the repo root and is registered in .claude/settings.json under the hooks.SessionStart array. Claude Code executes it on session open.

Two-Phase Execution

PhaseWhat happensWhy
Phase 1
(blocking)		 Prints a single JSON object to stdout containing hookSpecificOutput.additionalContext and hookSpecificOutput.reloadSkills: true Claude reads this before the conversation starts — it becomes the system-level context that governs the whole session
Phase 2
(background &)		 Syncs .claude/skills/* → ~/.claude/skills/, appends env vars to CLAUDE_ENV_FILE, runs npm install if package.json exists Non-blocking — session starts instantly, skills and env vars are ready within seconds

JSON Output Format

{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "... the full 4-step context string ...",
    "reloadSkills": true
  }
}

reloadSkills: true tells Claude Code to re-scan ~/.claude/skills/ and make all SKILL.md files available as slash commands immediately.

This is the exact content injected as additionalContext. Claude is blocked from responding to any user task until all 4 steps are confirmed complete.

How to Rebuild From Scratch

If the repo is lost, the hook is corrupted, or Claude starts a session without any context — use these steps to restore everything.

1. Restore the hook file

Create .claude/hooks/session-start.sh from the full source below and chmod +x it.

2. Register it in settings

In .claude/settings.json, ensure the hooks block includes:

{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/session-start.sh"
          }
        ]
      }
    ]
  }
}

3. Rebuild the skills

Each skill is a folder in .claude/skills/<name>/SKILL.md. The 36 skills currently installed are:

4. Restore the memory layer

The Google Drive folder 1uimIv6Uou7Ug0bYabz_P4YLr2LhVLiIU holds daily session notes in YYYY-MM-DD.md format. These are the long-term memory. The Drive folder ID never changes even if files are renamed.

5. Push and open a new session

Commit the restored hook + skills to branch main (or the active dev branch), push to Burst37/Space-Age-Skills, and start a new Claude Code session. The hook fires automatically and all 4 steps run before any task.

# ── BACKGROUND TASKS ──────────────────────────────────────────────────────────
(
  SKILLS_SRC="$CLAUDE_PROJECT_DIR/.claude/skills"
  SKILLS_DST="$HOME/.claude/skills"

  if [ -d "$SKILLS_SRC" ]; then
    mkdir -p "$SKILLS_DST"
    for skill_dir in "$SKILLS_SRC"/*/; do
      [ -d "$skill_dir" ] || continue
      skill_name=$(basename "$skill_dir")
      rm -rf "${SKILLS_DST:?}/$skill_name"

    done
